Clicking the Accept Terms button you will have a tutorial about how you can contribute.

Hi, welcome to the tutorial of this project. Below we show a screenshot of the application, explaining the different parts of it.


- General information about the user interface related bug, like Bug ID, Project, Product/Component and Bug Description.

- More information about the bug in Bugzilla bugtracking tool. A link back to this tutorial.

- Keep track of the amount of tasks completed in the project. You don`t need to complete all the tasks alone :-), there are other people contributing with the project too.

- General information about the bug taxonomy. Here you can see the taxonomy name and the description of the class you have selected as most representative of the bug.

- Here you can select the classes that may represent the bug in the taxonomy. Every taxonomy have some level of class hierarchy.

- You have three different types of answers:


  • You did find a classification at any level to the bug into the taxonomy. Please pay attention to the classes indexes, that appear on the left side of the name: they indicate the hierarchy of the classes, so for example, a second level class with index 31-SCND_LVL_CLASS_NAME cannot appear under a first level class with index 2-FRST_LVL_CLASS_NAME. Neither, for example, a third level class with index 212-THD_LVL_CLASS_NAME can appear under a second level class with index 22-SCND_LVL_CLASS_NAME. The application will alert you when something like that happen.

    Note that the amount of digits of the class index is correspondent to its level:

    • 1 digit -> first level class
    • 2 digits -> second level class
    • 3 digits -> third level class


  • You did not identify one class that may fit the bug into the taxonomy Please be sure you really did not find one before choose this option. Don`t worry to much in understand the bug, if you didn`t understand it after 1 minute, don`t waste your time with it.


  • You did not understand the bug description and so you are not able to classify it or there is a problem in the bug description. Please take a look at the Bugzilla link provided before choose this option.

Below we show a bug taxonomy, detailing the classes and levels.

The application is really simple. All you have to do is to identify into the taxonomy the bug classes considering the different levels of them, as shown above.

Observe that the class 1-GUI is correspondent to the 1-Class in the schema, as pointed out by the baloon, and the class 11-Components and Views is correspondent to the 11-Class in the schema, as pointed out by the baloon, and so on.

Note that there is an order that must be preserved in the classes indexes, so the class hierarchy is not broken at any time. The application will alert you in the case you try to submit one invalid classification.

Note also that if didn't find a complete path into the taxonomy in the three levels, you must complete the path with NIL options (class nodes) , so we can keep track of your choices. Empty answers ("-----") within a classification will not be accepted. The application will alert you in the case you try to submit one invalid classification.

That is it. Thanks again for your help and have fun!
